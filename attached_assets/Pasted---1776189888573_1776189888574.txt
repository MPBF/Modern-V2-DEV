# مواصفات أداة معالج تصميم الأكياس البلاستيكية

## الهدف

بناء أداة مستقلة عن نظام الإنتاج الحالي، مخصصة للعملاء أو موظفي المبيعات، تعمل كمعالج متعدد الخطوات (Wizard) يمكّن المستخدم من تكوين كيس بلاستيكي واقعي بصريًا وقابل للتصنيع، مع فرض قيود تمنع الاختيارات الخاطئة، ثم عرض نتيجة نهائية تشمل المواصفات الفنية والمعاينة البصرية ومحاكاة الطباعة.

هذه النسخة موجهة لوكيل Replit ليقوم بتطويرها كتطبيق مستقل.

---

## مخرجات الأداة

يجب أن تنتهي رحلة المستخدم إلى 3 مخرجات رئيسية:

1. **مواصفة فنية مختصرة ومنظمة**

   * نوع الكيس
   * العرض
   * الطول
   * الدخلات الجانبية
   * السماكة
   * نوع المادة
   * نوع المقبض
   * لون الكيس
   * هل الكيس مطبوع أو سادة
   * عدد ألوان الطباعة
   * ملاحظات القيود

2. **معاينة بصرية واقعية قدر الإمكان**

   * شكل الكيس ثلاثي الأبعاد أو شبه ثلاثي الأبعاد
   * لون المادة
   * شفافية تقريبية
   * محاكاة موضع الطباعة
   * عرض أمامي وخلفي ومائل إن أمكن

3. **ملف نتيجة نهائي**

   * بطاقة منتج قابلة للتصدير PDF أو صورة
   * JSON أو بيانات منظمة يمكن استخدامها لاحقًا للربط مع أنظمة أخرى

---

## نوع التطبيق المطلوب

تطبيق ويب مستقل بالكامل، غير مرتبط ببيئة الإنتاج الحالية.

### خصائصه

* واجهة عربية أولاً
* دعم إنجليزي لاحقًا
* يعمل على سطح المكتب والجوال
* سهل الاستخدام للعميل غير الفني
* يمنع الأخطاء قبل الوصول للخطوة النهائية
* لا يسمح بتركيبات غير صالحة

---

## النطاق الوظيفي الأساسي

الأداة يجب أن تغطي العناصر التالية بالمحددات الدنيا والعليا لكل عنصر:

* السماكة
* المقاس
* الدخلات الجانبية
* الطول إذا كان الكيس مطبوعًا
* الطول إذا كان الكيس سادة
* العرض
* المقبض
* المادة
* الطباعة
* لون الكيس نفسه
* عدد ألوان الطباعة
* محاكاة تصميم الطباعة
* عرض النتيجة

---

## المنهج العام

يجب بناء الأداة على **Rules Engine** واضح، بحيث تكون كل خطوة مرتبطة بقيود منطقية. لا تعتمد الأداة على مجرد إدخال حر من المستخدم، بل على اختيارات موجهة ومقيدة.

منطق العمل:

* المستخدم يختار نوع الكيس
* بناء على هذا النوع تظهر الخيارات الصالحة فقط
* كل قيمة يختارها المستخدم تؤثر على ما يليها
* إذا اختار قيمة تجعل الاختيارات التالية غير صالحة، يتم تحديث الخيارات تلقائيًا
* إذا تجاوز المستخدم حدًا معينًا تظهر رسالة خطأ أو تحذير أو اقتراح بديل

---

## أنواع الرسائل داخل النظام

يجب أن يدعم النظام 3 مستويات من التفاعل مع القيود:

### 1) منع كامل

يُستخدم عندما يكون الخيار غير قابل للتصنيع نهائيًا.
مثال:

* عرض خارج حدود هذا النوع من الأكياس
* عدد ألوان طباعة غير مدعوم
* مقبض غير متوافق مع نوع الكيس

### 2) تحذير

عندما يكون الخيار ممكنًا لكنه غير مفضل.
مثال:

* سماكة منخفضة مع مقاس كبير
* لون طباعة ضعيف الظهور على لون الكيس

### 3) اقتراح تلقائي

النظام يقترح قيمة أقرب للصواب.
مثال:

* تم اختيار طول مطبوع غير مناسب، فيقترح الطول الأدنى المقبول للطباعة

# مراحل المعالج

## الخطوة 1: اختيار نوع الكيس

### المدخلات

* كيس تيشرت
* كيس سفري/بدون يد
* كيس مقصوص
* كيس رول
* كيس نفايات
* كيس لحام سفلي
* كيس لحام جانبي

### المطلوب برمجيًا

* عند اختيار النوع يتم تحميل ملف القواعد الخاص به
* لا تُعرض إلا الحقول المطبقة على هذا النوع

### أمثلة قيود

* بعض الأنواع تدعم مقبضًا معينًا فقط
* بعض الأنواع لا تدعم الدخلات الجانبية
* بعض الأنواع لا تدعم الطباعة أو تدعمها بشروط

---

## الخطوة 2: تحديد ما إذا كان الكيس مطبوعًا أو سادة

### الخيارات

* سادة
* مطبوع

### التأثير

* إذا كان سادة يتم استخدام حدود الطول الخاصة بالسادة
* إذا كان مطبوعًا يتم استخدام حدود الطول الخاصة بالمطبوع
* إذا كان مطبوعًا تظهر حقول الطباعة وعدد الألوان والمحاكاة
* إذا كان سادة تُخفى إعدادات الطباعة بالكامل

---

## الخطوة 3: المادة

### الخيارات المقترحة

* HDPE
* LDPE
* LLDPE
* Blend
* ABA / متعدد الطبقات إن كان مطلوبًا في النسخة المتقدمة

### المطلوب

لكل مادة حدود خاصة أو توصيات خاصة في:

* السماكة الدنيا
* السماكة العليا
* المرونة البصرية
* الشفافية التقريبية
* مظهر السطح

### القيود

* ليس كل نوع كيس يدعم كل مادة
* بعض أنواع الطباعة قد تكون أقل وضوحًا على مادة معينة

---

## الخطوة 4: الأبعاد

هذه أهم خطوة ويجب فصلها بوضوح.

### الحقول

* العرض
* الطول
* الدخلات الجانبية
* السماكة

### منطق التحقق

يجب التحقق على مستويين:

#### أ) حدود كل حقل منفردًا

* min / max للسماكة
* min / max للعرض
* min / max للطول
* min / max للدخلات الجانبية

#### ب) حدود تركيبية بين الحقول

* علاقة العرض بالطول
* علاقة العرض بالدخلات الجانبية
* علاقة المقاس بالسماكة
* علاقة الطول بحالة الطباعة

### الحقول المطلوبة في ملف القواعد لكل نوع كيس

```json
{
  "thickness": { "min": 10, "max": 100, "unit": "micron" },
  "width": { "min": 10, "max": 120, "unit": "cm" },
  "length_plain": { "min": 20, "max": 200, "unit": "cm" },
  "length_printed": { "min": 25, "max": 180, "unit": "cm" },
  "side_gusset": { "min": 0, "max": 20, "unit": "cm" }
}
```

### ملاحظات مهمة

* `length_plain` و `length_printed` يجب أن يكونا منفصلين
* يمكن إظهار حقل الطول مرة واحدة للمستخدم، لكن التحقق يجب أن يعتمد على حالة الكيس مطبوع أو سادة
* إن كانت الدخلات الجانبية غير مدعومة يجب تثبيت القيمة = 0 مع إخفاء الحقل

## الخطوة 5: المقبض

### الخيارات تعتمد على نوع الكيس

* بدون مقبض
* تيشرت
* فتحة يد
* مقبض مقوى
* قص مخصص

### القيود

* بعض المقابض غير متاحة لبعض الأنواع
* بعض المقابض تتطلب عرضًا أو سماكة معينة
* بعض المقابض قد تؤثر على الطول الفعلي أو شكل المعاينة

### هيكل القواعد المقترح

```json
{
  "handle": {
    "allowed": ["none", "tshirt", "die_cut", "reinforced"],
    "rules": {
      "die_cut": {
        "min_width": 20,
        "min_thickness": 20
      },
      "reinforced": {
        "min_width": 25,
        "min_thickness": 30
      }
    }
  }
}
```

## الخطوة 6: لون الكيس

### الخيارات

* أبيض
* شفاف
* أسود
* أحمر
* أزرق
* أخضر
* أصفر
* ألوان مخصصة لاحقًا

### المطلوب

* التفريق بصريًا بين:

  * شفاف
  * نصف شفاف
  * معتم
* ربط اللون بتوصيات ظهور الطباعة

### قيود مهمة

* بعض ألوان الطباعة لن تكون واضحة على بعض ألوان الأكياس
* يجب حساب التباين البصري وإظهار تحذير إن كان ضعيفًا

---

## الخطوة 7: الطباعة

لا تظهر هذه الخطوة إلا إذا كان الكيس مطبوعًا.

### الحقول

* هل توجد طباعة على وجه واحد أو وجهين
* عدد ألوان الطباعة
* إدخال نصوص بسيطة
* رفع شعار أو صورة
* اختيار موضع الطباعة
* اختيار حجم الطباعة النسبي

### محددات عدد ألوان الطباعة

يجب أن تكون معرفة في القواعد بشكل min / max
مثال:

```json
{
  "print_colors": {
    "min": 1,
    "max": 6
  }
}
```

### قيود الطباعة

* إذا كان الكيس سادة، عدد ألوان الطباعة = 0
* إذا كان مطبوعًا، العدد يجب أن يقع ضمن الحدود المسموح بها
* يجب منع رفع تصميم لا يتناسب مع نسبة الأبعاد المتاحة للطباعة
* يجب تحذير المستخدم عند انخفاض جودة الصورة
* يجب تحذير المستخدم إذا كان لون الطباعة قريبًا جدًا من لون الكيس

### منطقة الطباعة الآمنة

يجب تعريف مستطيل أو مساحة آمنة للطباعة لكل نوع كيس، وعدم السماح بوضع عناصر خارجه.

مثال:

```json
{
  "print_area": {
    "front": { "x": 10, "y": 15, "width": 60, "height": 40, "unit": "percent" },
    "back": { "x": 10, "y": 15, "width": 60, "height": 40, "unit": "percent" }
  }
}
```

# قواعد التحقق المطلوبة

## 1) التحقق من السماكة

* لا تقل عن الحد الأدنى
* لا تتجاوز الحد الأعلى
* قد يزيد الحد الأدنى حسب العرض أو الطول أو نوع المقبض

## 2) التحقق من العرض

* ضمن حدود نوع الكيس
* قد يتغير الحد الأدنى أو الأعلى حسب المقبض أو المادة

## 3) التحقق من الطول

* إذا الكيس سادة استخدم `length_plain`
* إذا الكيس مطبوع استخدم `length_printed`
* أظهر رسالة واضحة عند تجاوز الحد

## 4) التحقق من الدخلات الجانبية

* ضمن الحدود إذا كانت مدعومة
* = 0 إذا غير مدعومة
* يمكن ربطها بعرض الكيس لمنع قيم غير متوازنة

## 5) التحقق من المقبض

* لا يُعرض إلا إذا كان مسموحًا
* يُرفض إذا لم تتحقق شروط العرض أو السماكة

## 6) التحقق من المادة

* يجب أن تكون من الخيارات المسموح بها لهذا النوع
* قد تؤثر على الشكل البصري والشفافية

## 7) التحقق من لون الكيس

* يجب أن يكون من قائمة الألوان المسموحة لهذا النوع
* يجب دعم تقييم تضاد اللون مع ألوان الطباعة

## 8) التحقق من عدد ألوان الطباعة

* إذا سادة: صفر
* إذا مطبوع: ضمن النطاق المحدد
* إذا تم تحميل تصميم يحتوي ألوانًا أكثر من الحد يمكن إظهار تحذير مبدئي فقط، وليس تحليلًا متقدمًا في النسخة الأولى

## 9) التحقق من تصميم الطباعة

* حجم ملف مناسب
* امتداده مدعوم
* دقة الصورة فوق حد معين
* لا يتجاوز منطقة الطباعة الآمنة

# منطق واجهة المستخدم

## مبادئ الواجهة

* كل خطوة قصيرة وواضحة

* الحقول الرقمية تدعم slider + input
* القيم تتحدث فورًا على المعاينة
* حفظ تلقائي للحالة أثناء التنقل بين الخطوات

# العرض البصري والواقعية

## النسخة الأولى المطلوبة

ليست محاكاة فيزيائية كاملة، لكن يجب أن تبدو مقنعة.

### المطلوب بصريًا

* نموذج كيس واضح حسب النوع
* لون خامة قريب من الواقع
* شفافية تقريبية لبعض المواد

* تطبيق الطباعة على واجهة الكيس
* دعم التكبير والتدوير

## خيارات التنفيذ

### الخيار الأفضل

* React + React Three Fiber + Three.js

### خيار أبسط وأسرع

* 2.5D Mockup باستخدام SVG أو Canvas مع منظور مزيف

### التوصية

ابدأ بـ **2.5D متقن** إذا كان الهدف سرعة التنفيذ، ثم أضف 3D لاحقًا.
لكن إذا كان وكيل Replit قادرًا على التنفيذ الجيد مباشرة، فالأفضل بناء المعاينة باستخدام React Three Fiber من البداية.

# قواعد UX مهمة جدًا

## 1) لا تعرض كل شيء مرة واحدة

المستخدم يجب أن يتحرك خطوة بخطوة.

## 2) لا تترك الحقول الرقمية مفتوحة بلا حدود

كل حقل يظهر معه الحد الأدنى والأعلى بوضوح.
مثال:

* العرض: من 20 إلى 70 سم
* السماكة: من 10 إلى 40 ميكرون

## 3) عند تغيير نوع الكيس

إذا أصبحت بعض القيم الحالية غير صالحة:

* إما تصفيرها
* أو تعديلها تلقائيًا مع إشعار المستخدم

## 4) في خطوة الطباعة

إذا اختار سادة سابقًا ثم عاد وجعلها مطبوعة:

* تظهر الحقول الجديدة بقيم ابتدائية مناسبة

# نص أمر مختصر جاهز لوكيل Replit

أنشئ تطبيق ويب مستقل باللغة العربية وباتجاه RTL عبارة عن معالج متعدد الخطوات لتصميم الأكياس البلاستيكية. يجب أن يسمح للمستخدم باختيار نوع الكيس، حالته مطبوع أو سادة، المادة، العرض، الطول، الدخلات الجانبية، السماكة، المقبض، لون الكيس، وعدد ألوان الطباعة. يجب أن يطبق حدودًا دنيا وعليا وقواعد توافق على كل من: السماكة، المقاس، الدخلات الجانبية، الطول إذا كان الكيس مطبوعًا، الطول إذا كان سادة، العرض، المقبض، المادة، الطباعة، لون الكيس، وعدد ألوان الطباعة. يجب أن يمنع التركيبات غير القابلة للتصنيع ويظهر أخطاء وتحذيرات واقتراحات. يجب أن يتضمن محاكاة بصرية قريبة من الواقع لشكل الكيس مع إسقاط تصميم الطباعة عليه، وأن يعرض في النهاية صفحة نتيجة تحتوي على ملخص المواصفات، حالة الصلاحية، التحذيرات، وصورة نهائية قابلة للتصدير PDF وصورة وJSON. استخدم React + TypeScript + Tailwind + Zustand + React Hook Form + Zod، وافصل محرك القواعد عن الواجهة، وخزن القواعد في JSON ليسهل تعديلها لاحقًا.
